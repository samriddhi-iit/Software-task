# Task -2

- Obtain the world map
    
    Looping and adding, reviewing which points are near the obstacles, and marking them green and red based on that.
    
    This is stored in memory for later access.
    
    ![image.png](Task%20-2/image.png)
    
- Triangulate the free area and treat the centroids of the triangles obtained as waypoints.
- Pure Pursuit model
    
    The path has a sequence of 2D waypoints to follow. The vehicle knows that sequence.
    
    The vehicle can localise itself in the given waypoint system of reference.
    
    The goal is to follow these wavepoints.
    
    - Maths behind the model
        
        Steering angle is inversely proportional to the curvature of the arc.
        
        ![image.png](Task%20-2/image%201.png)
        
    - Tuning L parameter (look-ahead distance)
        
        The look-ahead distance (L) is a parameter in the purse pursuit algorithm:
        A smaller L leads to more aggressive maneuvers.
        A larger L leads to a smoother trajectory but with higher tracking errors, which can lead to close approaches to obstacles.
        
    
    Every time we have a new pose of the car:
    
    1. Get the closest waypoint to the vehicle.
    2. Go up towards that waypoint with the calculated steering angle.
    3. Localize to find the new pose.

This program implements an autonomous agent that navigates using a grid-based local path planning approach.

## Safe Grid Generation

- 20×20 grid in range [-1,1]
- Remove unsafe points near obstacles
- Store safe points

---

## Goal Definition

- Fixed goal: (-0.9, -0.7)
- Stop when close

---

## Target Selection

- Choose forward points
- Minimize distance to goal
- Store as current target

---

## Control Logic

- Steering using `atan2`
- Clamp steering
- Adjust speed based on turning

- Run simulation loop
